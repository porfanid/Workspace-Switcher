Workspace Switcher - Usage Examples

Integrating with xbindkeys

xbindkeys is a tool that allows you to bind actions to keyboard and mouse events. You can use it to integrate Workspace Switcher with your mouse for convenient workspace switching.

Installing xbindkeys

First, you need to install xbindkeys if you haven’t already. On Ubuntu, you can install it using:

sudo apt-get install xbindkeys

Configuring xbindkeys

  1. Open the xbindkeys Configuration File

    The configuration file is usually located at ~/.xbindkeysrc. Open it with your preferred text editor:

    nano ~/.xbindkeysrc

  2. Add Mouse Bindings

    Add the following lines to bind mouse buttons to the workspace switching commands. For example, you might want to bind the middle mouse button to switch to the next workspace and the right mouse button to switch to the previous workspace:

    "/usr/bin/workspaces next" b:8 "/usr/bin/workspaces prev" b:9

    Replace b:8 and b:9 with the mouse buttons you want to use. You can use tools like xev to identify the button numbers.

  3. Apply the Configuration

    After editing the configuration file, apply the changes with:

    xbindkeys

    Alternatively, you may need to restart xbindkeys or your X session for the changes to take effect.

Summary

By integrating Workspace Switcher with xbindkeys, you can seamlessly switch between workspaces using mouse buttons. This setup enhances productivity and provides a more intuitive way to manage multiple workspaces.

For more advanced configurations or troubleshooting, refer to the xbindkeys documentation.
